# App Repo Blueprint: Build and Publish Artifact

This blueprint defines the contract between app and infra repositories.

## Goals

- Build, test, and security scan in app repo CI.
- Push service images to Amazon ECR from release workflow.
- Publish an immutable release manifest using image digests.
- Open PR to infra repo to bump target environment version.

## Workflows

- CI workflow: [.github/workflows/app-ci.yml](../.github/workflows/app-ci.yml)
  - Trigger: pull_request to main and push to main.
  - Runs unit tests, integration tests, image build, and security scan.
  - Does not deploy infrastructure.

- Release artifact workflow: [.github/workflows/release-artifact.yml](../.github/workflows/release-artifact.yml)
  - Trigger: release tags v* or manual workflow_dispatch.
  - Builds and pushes service images: catalog, cart, orders, checkout, ui to all configured failover regions.
  - Resolves image digests in every region and generates a multi-region release manifest.
  - Fails fast if any service image is missing in any configured region.
  - Resolves infra version file automatically using environment-to-path repository variable mapping.
  - Optionally signs manifest if cosign secrets are configured.
  - Uploads manifest as GitHub build artifact.
  - Opens PR in infra repo to update target environment version file.

## Data Contract Between App and Infra Repos

App repo publishes one canonical manifest file generated by [scripts/generate-release-manifest.sh](../scripts/generate-release-manifest.sh):

```yaml
release:
  id: 2026.04.07-rc1
  git_sha: 123abc456def
  created_at: 2026-04-07T03:00:00Z
  regions:
    - ap-southeast-1
    - ap-northeast-1
services:
  catalog:
    regions:
      ap-southeast-1:
        repository: 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com/retail-store-sample-catalog
        tag: 1.4.0
        digest: sha256:...
      ap-northeast-1:
        repository: 123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/retail-store-sample-catalog
        tag: 1.4.0
        digest: sha256:...
  cart:
    regions:
      ap-southeast-1:
        repository: 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com/retail-store-sample-cart
        tag: 1.4.0
        digest: sha256:...
      ap-northeast-1:
        repository: 123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/retail-store-sample-cart
        tag: 1.4.0
        digest: sha256:...
```

Infra repo must consume this manifest as the source of truth for deployment.

## PR Automation to Infra Repo

Release workflow uses bot or GitHub App token to open a PR in infra repo with:

- Updated target environment version file.
- Release metadata: release id, git sha, and workflow run URL.
- Manifest file copied into infra repo for deterministic deployment.

Policy:

- Do not auto-merge prod release PR.
- test/staging may auto-merge only if infra repo policy allows it.

## Required Repository Secrets and Variables

Secrets:

- AWS_ROLE_ARN
- AWS_ECR_REGISTRY
- INFRA_REPO (format: org/repo)
- INFRA_REPO_TOKEN
- COSIGN_PRIVATE_KEY (optional)
- COSIGN_PASSWORD (optional)

Variables:

- AWS_REGIONS_JSON (optional, default ["ap-southeast-1","ap-northeast-1"])
- INFRA_BASE_BRANCH (optional, default main)
- INFRA_VERSION_FILE_MAP_JSON (optional)

Example INFRA_VERSION_FILE_MAP_JSON:

```json
{"test":"environments/test/app-release.yaml","staging":"environments/staging/app-release.yaml","prod":"environments/prod/app-release.yaml"}
```

The workflow selects the path by target environment and fails fast if the map is invalid or missing a key.

Example AWS_REGIONS_JSON:

```json
["ap-southeast-1","ap-northeast-1"]
```

## IAM and Security for App Repo

Use GitHub OIDC role with minimum permissions required for ECR push and digest lookup.

Example IAM policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "EcrAuth",
      "Effect": "Allow",
      "Action": [
        "ecr:GetAuthorizationToken"
      ],
      "Resource": "*"
    },
    {
      "Sid": "EcrPushAndDescribe",
      "Effect": "Allow",
      "Action": [
        "ecr:BatchCheckLayerAvailability",
        "ecr:CompleteLayerUpload",
        "ecr:DescribeImages",
        "ecr:InitiateLayerUpload",
        "ecr:PutImage",
        "ecr:UploadLayerPart"
      ],
      "Resource": "arn:aws:ecr:ap-southeast-1:<account-id>:repository/retail-store-sample-*"
    }
  ]
}
```

Do not grant Terraform apply or Kubernetes admin permissions to app repo workflows.
